# Visions15

Desktop-приложение для работы с изображениями деталей: сканирование с камеры, сбор датасета по классам, выгрузка выбранных классов на сервер и обновление локальной модели.

Интерфейс написан на PySide6, работа с камерой и ArUco-маркерами выполняется через OpenCV.

## Возможности

- Сканирование изображения с камеры и сохранение результата в `captures/scanned`.
- Определение рабочей области по ArUco-маркерам и перспективное выравнивание кадра.
- Ведение списка деталей в `details.json`: название класса и артикул.
- Добавление изображений в датасет вручную снимками или записью с заданным интервалом.
- Формирование `captures/dataset/metadata.json` по сохраненным классам.
- Выгрузка выбранных классов на сервер архивом `tar.gz`.
- Перенос выгруженных классов в `captures/backup_dataset`.
- Проверка соединения с сервером и обновление локальных файлов модели.

## Требования

- Python 3.10 или новее.
- Подключенная камера, доступная через OpenCV.
- ArUco-маркеры словаря `DICT_4X4_50` с ID `0`, `1`, `2`, `3` для определения рабочей области.
- Доступ к API-серверу для выгрузки датасетов и обновления модели.

Зависимости перечислены в `requirements.txt`:

```text
PySide6
PyYAML
opencv-python
requests
```

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Создайте файл `.env` на основе `.env.example` и укажите актуальные значения:

```env
API_URL="https://example.com/api/v1"
API_KEY="your-api-key"
UNLOAD_TIME_1="17:00"
UNLOAD_TIME_2="08:00"
```

## Запуск

```powershell
.\.venv\Scripts\Activate.ps1
python main.py
```

После запуска откроется окно `IR AI` с основными разделами:

- `Сканирование` - сделать снимок, применить выравнивание по ArUco-маркерам и открыть сохраненное изображение.
- `Добавить деталь` - создать или выбрать класс детали, затем снять изображения для датасета.
- `Выгрузка` - выбрать классы из датасета и отправить архив на сервер.
- `Настройки` - сохранить ключ API, часы выгрузки, параметры камеры, проверить сервер и обновить модель.

## Конфигурация камеры

Параметры камеры хранятся в `camera_config.json`.

```json
{
  "device_index": 0,
  "mjpg": false,
  "fps": 1,
  "width": 1920,
  "height": 1080,
  "scan_warmup_seconds": 1.0,
  "scan_max_wait_seconds": 5.0
}
```

Поддерживаемые поля:

- `device_index` - индекс камеры для `cv2.VideoCapture`.
- `mjpg` - включить MJPG-формат камеры.
- `fps` - частота чтения кадров.
- `width`, `height` - запрашиваемое разрешение камеры.
- `capture_interval` - интервал записи изображений датасета в секундах. Если поле отсутствует, используется значение по умолчанию `1`.
- `scan_warmup_seconds` - время прогрева камеры перед сканированием.
- `scan_max_wait_seconds` - максимальное ожидание стабильного кадра.

Часть параметров можно изменить из раздела `Настройки`.

## Данные проекта

Основные файлы и каталоги:

```text
.
|-- main.py
|-- buttons.yaml
|-- camera_config.json
|-- details.json
|-- captures/
|   |-- scanned/
|   |-- dataset/
|   |   |-- metadata.json
|   |   `-- <class_name>/images/
|   `-- backup_dataset/
|-- model/
|   |-- manifest.json
|   `-- model.pt
|-- screens/
`-- services/
```

`details.json` хранит соответствие названия детали и артикула:

```json
{
  "detail_name": "12345"
}
```

`captures/dataset/metadata.json` создается при сохранении изображений датасета. Формат:

```json
{
  "schema_version": "1.0",
  "dataset_update": {
    "name": "dataset-25-06-2026"
  },
  "classes": {
    "detail_name": {
      "article": "12345",
      "directory": "detail_name",
      "images_count": 10
    }
  }
}
```

Названия классов не должны быть пустыми и не могут содержать символы:

```text
< > : " / \ | ? *
```

## Выгрузка датасета

При выгрузке выбранных классов приложение:

1. Читает `captures/dataset/metadata.json`.
2. Создает архив `captures/dataset/<dataset_name>.tar.gz`.
3. Отправляет архив на endpoint:

```text
POST <API_URL>/uploads/archive
```

4. Передает ключ в заголовке:

```text
X-API-Key: <API_KEY>
```

5. После успешной отправки переносит выгруженные папки классов в `captures/backup_dataset` и удаляет эти классы из текущего `metadata.json`.

## Серверные endpoint'ы

Приложение использует следующие endpoint'ы:

- `GET <API_URL>/health` - проверка соединения.
- `POST <API_URL>/uploads/archive` - загрузка архива датасета.
- `GET <API_URL>/model/manifest` - получение manifest текущей модели.
- `GET <API_URL>/model/latest` - скачивание архива последней модели.

Для загрузки модели архив должен содержать файлы:

```text
manifest.json
model.pt
```

После обновления они сохраняются в каталог `model`.

## Структура кода

- `main.py` - главное окно, навигация между экранами, обработчики пользовательских действий.
- `screens/` - PySide6-экраны приложения.
- `services/camera_service.py` - работа с камерой, сохранение кадров, структура датасета и metadata.
- `services/perspective_warp_service.py` - поиск ArUco-маркеров и перспективное преобразование изображения.
- `services/upload_service.py` - создание архивов, чтение `.env`, выгрузка датасета на сервер.
- `services/model_service.py` - проверка и обновление файлов модели.

## Примечания

- Файл `.env` не должен попадать в репозиторий.
- В текущем интерфейсе результат сканирования в таблице заполняется заглушками `null`; сохранение изображения уже выполняется.
- Для корректного сканирования четыре ArUco-маркера должны быть видны в кадре.
